# 国际化 i18n

## 适用范围
- 所有返回给前端的 `message`
- 请求参数校验失败提示
- 全局异常处理中返回给前端的错误提示
- schema 自定义校验提示

## 编码规范
- 使用 `from flask_babel import gettext as _`
- 所有需要翻译的文案统一写成 `_("English message")`
- `msgid` 统一使用英文原文，不额外维护 `message_id`
- 不要对动态拼接后的整句做翻译

推荐写法：

```python
message = _("Login successful")
message = _("Invalid username or password")
raise BadRequest(_("Request body must be valid JSON"))
raise BadRequest(_("Missing uploaded file: %(name)s", name=name))
```

不推荐写法：

```python
_(f"{name} created successfully")
```

## 开发流程
1. 在代码中新增或修改 `_("...")` 文案
2. 提取最新翻译字符串
3. 更新语言目录下的 `.po` 文件
4. 编辑 `.po` 文件中的 `msgstr`
5. 编译生成 `.mo` 文件

## Babel 命令

### 首次初始化方案
1. 先把 `messages.pot` 生成到项目根目录
2. 再使用 `pybabel init` 初始化 `app/translations`

```bash
pybabel extract -F babel.cfg -o messages.pot .
pybabel init -i messages.pot -d app/translations -l zh
```

说明：
- 这是最省事的初始化方式
- 因为第一次执行时，`app/translations/` 目录可能还不存在
- 当前项目继续把 `messages.pot` 放在根目录也完全可以

### 后续更新方案
如果以后想把 i18n 文件集中管理，也可以在 `app/translations/` 已经存在后，把 `messages.pot` 生成到翻译目录中：

```bash
pybabel extract -F babel.cfg -o app/translations/messages.pot .
pybabel update -i app/translations/messages.pot -d app/translations
pybabel compile -d app/translations
```

当前项目如果继续使用根目录 `messages.pot`，后续维护也可以这样执行：

```bash
pybabel extract -F babel.cfg -o messages.pot .
pybabel update -i messages.pot -d app/translations
pybabel compile -d app/translations
```

## 翻译文件位置
- 模板文件：`messages.pot` 或 `app/translations/messages.pot`
- 中文翻译：`app/translations/zh/LC_MESSAGES/messages.po`
- 编译结果：`app/translations/zh/LC_MESSAGES/messages.mo`

## 运行时语言选择
- 当前项目优先使用浏览器 `Accept-Language`
- 如果后续需要支持前端手动切换语言，可以扩展 `app/i18n.py`
